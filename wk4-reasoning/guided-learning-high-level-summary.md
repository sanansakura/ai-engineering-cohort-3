# Week 4 Guided Learning — High-Level Summary

**Reasoning Models** are LLMs that generate intermediate "thinking" traces before producing a final answer, exploring multiple reasoning paths before committing to a response. Most top-ranked models today (o3, o4 mini, DeepSeek R1) are reasoning models. Non-reasoning models (GPT-4o, GPT-4.5) respond immediately without a thinking phase but may still show some reasoning behavior from their training data.

- **Building Reasoning Capability — Two Buckets**
  - **Inference-Time Scaling** — Spend more compute at generation time to improve outputs without changing model parameters. The model thinks longer and harder at the point of use.
    - **Chain-of-Thought (CoT) Prompting** — Guide the model to reason step by step, either by providing worked examples in the prompt (few-shot CoT) or simply adding "Let's think step by step" (zero-shot CoT). Even non-reasoning models like LLaMA 3 go from wrong answers to correct answers when given CoT examples.
    - **Self-Consistency / Parallel Sampling** — Generate N complete responses in parallel and pick the best one via majority voting (for verifiable answers) or reward model scoring (for open-ended tasks). Consistently outperforms single-generation CoT across benchmarks.
    - **Tree of Thoughts** — Treat generation as a tree search: generate intermediate steps, score them, expand promising branches, and prune bad ones early. More compute-efficient than self-consistency because it avoids completing low-quality branches.
    - **Sequential Revision** — Feed the model its own output and ask it to revise iteratively for k rounds. Better for easy/medium problems where a quick fix suffices; parallel sampling is better for hard problems where you need diverse solution approaches.
  - **Training-Time Scaling** — Modify the model's parameters so it internalizes reasoning capability permanently.
    - **Fine-Tuning on CoT Data** — Generate reasoning traces via prompting, filter for correct answers, and use those question + reasoning pairs to fine-tune the model. Special tokens (`<think>`, `</think>`) delineate reasoning from final output. The process is iterative — each round of fine-tuning produces a model that generates better training data for the next round.
    - **Outcome Reward Modeling (ORM)** — Train a reward model to score the final answer's correctness, then use those scores to continue fine-tuning the LLM. Limitation: ORM ignores intermediate reasoning steps entirely, so it can't detect flawed reasoning that happens to reach a correct answer.
    - **Process Reward Modeling (PRM)** — Scores every intermediate reasoning step individually rather than just the final answer, giving much more granular feedback. PRM has a powerful dual use: it improves training (richer signal) and also guides inference-time search (pruning bad reasoning paths in real time). The main challenge is that labeling individual steps is hard to automate — a DeepMind paper proposes Monte Carlo rollouts as a scalable solution.
- **Self-Correction** — Methods for the model to detect and fix its own mistakes through iterative revision (generate → evaluate → revise → repeat).
  - Can happen at inference time (prompt tricks) or training time (RL is significantly more effective than SFT for teaching self-correction)
  - **Intrinsic** self-correction uses only the model's own judgment; **extrinsic** self-correction leverages external signals like reward models, code execution results, or tool outputs — extrinsic is far more practical and effective in real-world settings
- **Internalizing Search (Meta-CoT)** — For complex problems, a single linear chain of thought isn't enough; the model needs to try different approaches and backtrack. Meta-CoT trains on data containing sequences of "latent thoughts" (z1, z2, ... zk) — failed attempts followed by the correct solution — so the model learns to search, backtrack, and explore adaptively. Real models like o1 and DeepSeek R1 show clear backtracking patterns ("But wait...", "Actually..."). The model also learns adaptive thinking depth: quick answers for simple questions, extensive search for complex ones.
- **Deep Research** — A practical multi-agent application that combines everything: a reasoning model as the core agent, multiple sub-agents performing independent web research tasks, and a manager agent that coordinates and compiles a comprehensive final report. OpenAI's deep research feature takes tens of minutes and can now visually browse web pages. Project 4 builds a "deep research light" version of this system.
